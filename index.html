Informe técnico de plan de trabajo para la construcción de software

Las herramientas de control de versiones permiten gestionar los cambios en los archivos de un proyecto a lo largo del tiempo. Son fundamentales para el desarrollo de software, ya que permiten mantener un historial claro de modificaciones, facilitar el trabajo colaborativo y revertir errores cuando sea necesario.

Su uso es crucial en proyectos modernos, ya que optimizan la productividad del equipo y reducen riesgos de pérdida de información o conflictos entre versiones.

Diferencias entre versionamiento local y remoto

El versionamiento local se limita al entorno del desarrollador, es decir, los cambios se registran y administran solo en su computadora personal. Aunque útil, no permite el trabajo colaborativo a gran escala.

El versionamiento remoto, por otro lado, permite almacenar el proyecto en servidores externos (como GitHub, GitLab o Bitbucket), lo que permite compartirlo, colaborar en tiempo real, mantener respaldo y trabajar desde cualquier lugar.

Herramientas de versionamiento

A continuación, se describen algunas herramientas de control de versiones más utilizadas:

Git

Es un sistema de control de versiones distribuido, gratuito y de código abierto. Permite el trabajo colaborativo y mantiene un historial detallado del proyecto.

GitLab

Plataforma basada en Git que facilita la colaboración entre desarrolladores, la integración continua y la entrega continua (CI/CD).

Bitbucket

Herramienta de Atlassian que permite gestionar repositorios privados con integración a Jira. Ideal para equipos de desarrollo ágil.

Apache Subversion (SVN)

Sistema de control de versiones centralizado, aún usado en entornos corporativos por su simplicidad y estructura rígida.

Herramienta seleccionada para desarrollar

Para este proyecto se ha decidido utilizar Git como herramienta de control de versiones y GitHub como plataforma de alojamiento remoto. Esta combinación ofrece una solución robusta y ampliamente utilizada en la industria.

Git permite trabajar de forma distribuida, lo que significa que cada desarrollador tiene una copia completa del historial del proyecto. Además, su velocidad, flexibilidad y soporte comunitario lo convierten en la mejor opción actual.

GitHub, por su parte, facilita la colaboración remota, el control de versiones en línea, el manejo de issues, pull requests y revisión de código entre compañeros.
